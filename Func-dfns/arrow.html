Arrow Functions

They allow a short syntax for writing functions expressions.
You don't need the function keyword, and the return keyword and the carly brackets.

Arrow functions are not supported in IE11 or earlier.

Arrow functions do not have their own this.
They are not well suited for defining object methods.
Arrow functions are not hoisted. They must be defined before they are used.
Using const is safer than using var, because functions expression is always constant value.
You can only omit the return keyword and the carly brackets if the function is a single statement.
Because of this it might be a good habit to always keep them.
